# Loan Kit Grouping Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow loans to include entire kits (with a quantity), persist kit attribution on each loan item, and display loan items grouped by kit in both the create flow and loan list.

**Architecture:** A2 schema — three nullable columns (`kit_id`, `kit_name`, `kit_quantity`) added to `loan_items`. `kit_id` / `kit_name` / `kit_quantity` are identical across all rows belonging to the same kit within a loan. Frontend cart is restructured from a flat `CartItem[]` to a typed union `CartEntry[]` (`KitCartEntry | AssetCartEntry`). The create-loan drawer Step 1 gains a Kits tab alongside the existing Assets tab. Expanded loan rows and the edit modal reconstruct grouping from the stored kit fields.

**Tech Stack:** C# ASP.NET Core (backend), React + Ant Design (frontend), PostgreSQL (DB migration script applied manually)

function loadCart(userId: string | undefined): CartEntry[] {
  if (!userId) return [];
  try {
    const raw = localStorage.getItem(`cart_${userId}`);
    if (!raw) return [];
    const parsed = JSON.parse(raw) as CartEntry[];
    // Discard old format (no type discriminant)
    if (parsed.length > 0 && !('type' in parsed[0])) return [];
    return parsed;
  } catch { return []; }
}

function saveCart(userId: string | undefined, items: CartEntry[]) {
  if (!userId) return;
  localStorage.setItem(`cart_${userId}`, JSON.stringify(items));
}
